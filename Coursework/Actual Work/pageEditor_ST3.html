Feedback Page

Technical Description

The Feedback Page is an essential part of the website where users can provide input regarding their experience. For the structure, I used basic HTML elements such as <form>, <input>, and <textarea> to allow users to submit their feedback easily. These tags were chosen for their specific roles in collecting and submitting data. For styling, I relied heavily on CSS for visual consistency across the page, using properties like padding, margin, and background-color to enhance the user experience. These choices were aimed at making the page easy to navigate, ensuring it was both functional and visually appealing.

Accessibility

To ensure the page is accessible to all users, I focused on providing alternative text for form elements and labels. For example, I included descriptive <label> tags for input fields to improve form navigation, especially for screen reader users. Additionally, color contrast was optimized, ensuring readability for users with visual impairments.

Team Page

Technical Description

The Team Page provides detailed information about the members involved in the project. I implemented the layout using HTML5 semantic elements like <section>, <article>, and <figure>, which ensure the content is both structured and easily identifiable. These elements were styled using CSS to create a clean, organized look with interactive thumbnails. The use of hover effects in CSS enhances the visual appeal and interactivity of the page. This combination of HTML and CSS ensures both functionality and a visually engaging experience for users.

Challenges with JavaScript

While implementing the hover and expanding functionality for the team members, I faced challenges in achieving smooth and interactive behavior. Initially, using pure CSS hover effects didn’t work well for the expanding functionality, as it lacked control over more complex transitions and didn’t adapt well to certain devices.

To fix this, I used JavaScript to control the dynamic height adjustments for the expandable sections of each team member’s profile. This was necessary because the pure CSS solution caused layout issues, especially for different screen sizes. The JavaScript functions were used to add and remove CSS classes that controlled the size of the profile cards on hover.

Additionally, I had trouble ensuring that the hover effect was responsive across browsers, as some browsers rendered the transitions differently. I addressed this by testing the JavaScript code for cross-browser compatibility, ensuring it would work consistently across Chrome, Firefox, and Safari.

Accessibility

I prioritized accessibility by ensuring all images on the page have alt attributes that describe their content, which benefits screen reader users. Additionally, I used a clear and simple navigation structure to make it easier for users with disabilities to navigate the page.

Content Page

Technical Description

The Content Page serves as the main informational page of the site, introducing the project's objectives and providing crucial details. I employed a straightforward structure with <article> tags to encapsulate sections of content and <header> tags for section titles. The page's layout was styled with CSS, using properties like flexbox to ensure responsiveness across devices. The decision to use CSS Grid also allowed for a clean, organized presentation of text and images.

Accessibility

I addressed accessibility by ensuring that the text is readable with high contrast and providing alternative text for images. I also included keyboard navigability for easier access to all interactive elements.

Challenges and Lessons Learned

During the development of the pages, one of the main challenges I faced was ensuring the responsiveness of the layout across different screen sizes. Initially, some elements weren’t aligning as expected, especially on mobile devices. To resolve this, I used CSS media queries to adjust the layout dynamically based on the screen width, which improved the user experience on smaller devices. From this, I learned the importance of planning for mobile-first design and how it can significantly enhance the accessibility of the page.

Compliance

The pages I developed are fully compliant with web accessibility guidelines and the Janet regulations governing web page publication. I made sure to implement valid HTML5 and CSS standards throughout the design process, ensuring the website functions correctly across all browsers and devices. The project adheres to WCAG 2.1 accessibility guidelines to provide an inclusive experience for users with disabilities.
